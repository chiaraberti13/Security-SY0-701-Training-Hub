Leggere gli indicatori: dalla sintomatologia alla diagnosi

L'obiettivo 2.4 non chiede di sapere come costruire un attacco, ma di saper riconoscere i suoi indicatori (indicators) — i segnali che rimangono nel sistema dopo o durante un'attività malevola. Come un medico che diagnostica una malattia dai sintomi senza averla vista formarsi, un analista di sicurezza deve leggere log, traffico di rete, comportamento dei processi e anomalie di sistema per identificare cosa sta accadendo.

Le domande d'esame presentano sempre uno scenario: un log, un output di monitoring, un comportamento anomalo. Il compito è classificare correttamente l'attività. Il metodo: identificare prima la categoria dell'attacco (malware, rete, applicativo, crittografico, password), poi il tipo specifico.

Attacchi malware (Malware Attacks)

Ransomware

Il ransomware è malware che cifra i file della vittima e richiede un riscatto per fornire la chiave di decifratura. È la minaccia economicamente più impattante degli ultimi anni. L'indicatore caratteristico è la presenza di file con estensioni anomale (es. .ryk, .locked, .encrypted) e di un file di testo o immagine con le istruzioni per il pagamento del riscatto.

Indicatori tipici
File con estensione modificata in modo non riconosciuto (SCREEN_303: .ryk). Nota di riscatto (ransom note) sul desktop o in ogni cartella cifrata. CPU e I/O disco elevati durante la fase di cifratura. Accesso massiccio e sequenziale ai file in breve tempo. Comunicazione outbound verso indirizzi sospetti (C2 per la chiave).
Ransomware-as-a-Service (RaaS)
Il modello operativo del crimine organizzato: il gruppo sviluppa il ransomware e lo affitta ad affiliati che si occupano della distribuzione, dividendo il riscatto. Ha democratizzato gli attacchi ransomware aumentandone il volume.
Double extortion
Tecnica moderna: l'attaccante non solo cifra i dati ma li esfil prima, minacciando di pubblicarli se il riscatto non viene pagato — anche se la vittima ripristina dal backup.
EXAM TIP — Ransomware: l'estensione è la chiave
"All user workstations and servers are displaying a message associated with files containing extension .ryk" → Ransomware (SCREEN_303). L'estensione anomala + messaggio di riscatto = indicatori diagnostici del ransomware. Non è un virus (che non cifra necessariamente), non è spyware (che non blocca i file).

Trojan (Cavallo di Troia)

Un Trojan (Trojan Horse) è malware mascherato da software legittimo o utile. L'utente lo installa volontariamente credendo si tratti di un'applicazione desiderabile (gioco, utility, codec, crack di software a pagamento). Una volta installato, esegue funzionalità malevole nascoste: aprire backdoor, scaricare altro malware, registrare l'attività dell'utente.

Il Trojan non si replica autonomamente (si distingue dal worm per questo) e non infetta altri file (si distingue dal virus). Richiede che l'utente lo installi deliberatamente, sfruttando l'inganno. Lo scenario SCREEN_372 (software scaricato da forum online che genera traffico outbound su porta non comune) descrive un Trojan con backdoor integrata.

Worm

Un worm è malware che si replica e si propaga autonomamente attraverso la rete senza bisogno di intervento umano: non richiede che l'utente clicchi su un link, apra un allegato o installi qualcosa. Sfrutta vulnerabilità di servizi di rete per infettare sistemi raggiungibili. È questa caratteristica di auto-propagazione che lo distingue da tutti gli altri malware.

Indicatori tipici
Traffico di rete elevato e anomalo su porte di servizi specifici (SCREEN_713: TCP 445 = SMB, tipico di worm come WannaCry). Latenza di rete elevata. Sistemi che tentano di connettersi a molti altri host in sequenza. Nessun intervento utente rilevato prima dell'infezione.
Vettori tipici
Vulnerabilità nei servizi di rete esposti (SMB, RDP, SSH). Una volta entrato in un sistema, il worm scansiona automaticamente altri host raggiungibili e ripete il processo.
ATTENZIONE — Worm vs Virus vs Trojan
Worm: si propaga autonomamente, nessun intervento umano, sfrutta vulnerabilità di rete. Indicatore: alto traffico di rete, molti sistemi infettati simultaneamente (SCREEN_003, SCREEN_713).
Virus: si replica infettando altri file, richiede esecuzione del file infetto da parte dell'utente.
Trojan: si maschera da software legittimo, richiede installazione volontaria, non si replica autonomamente.

Spyware e Keylogger

Lo spyware è malware che raccoglie informazioni sull'utente e le trasmette a terzi senza consenso: cronologia di navigazione, abitudini d'uso, credenziali, dati personali. È spesso installato insieme ad altro software (bundling) o tramite siti web compromessi. Difficile da rilevare perché opera silenziosamente in background.

Il keylogger è un tipo specializzato di spyware che registra ogni tasto premuto dall'utente, catturando credenziali, messaggi privati, numeri di carta di credito e qualsiasi altra informazione digitata. Può essere software (processo che intercetta gli eventi tastiera) o hardware (dispositivo fisico tra tastiera e PC).

Indicatori keylogger software
Processo sconosciuto in esecuzione con permessi elevati. Traffico outbound verso IP sospetti con dati cifrati. Degradazione delle prestazioni. Logs di sistema che mostrano accesso a API di hook della tastiera.
Indicatori keylogger hardware
Dispositivo fisico tra il connettore della tastiera e la porta USB/PS2 del PC. Rilevabile solo con ispezione fisica del cablaggio.

Virus · Bloatware · Rootkit

Virus
Malware che si replica infettando altri file eseguibili. Richiede che l'utente esegua il file infetto per attivarsi e propagarsi. Non si diffonde autonomamente sulla rete (a differenza del worm). Indicatore: hash di file eseguibili cambiato senza patch applicate.
Bloatware
Software indesiderato pre-installato dai produttori (tipicamente su PC nuovi) o installato come bundle. Non sempre malevolo, ma consuma risorse, riduce le prestazioni e può introdurre vulnerabilità. L'indicatore è la presenza di applicazioni non richieste che si avviano automaticamente.
Rootkit
Malware progettato per nascondere la sua presenza e quella di altri malware al sistema operativo e ai tool di sicurezza. Modifica le API di sistema per restituire risultati alterati: se cerchi il processo malevolo, il rootkit lo nasconde dall'elenco dei processi. Indicatore classico: hash di file di sistema cambiato senza patch recenti (SCREEN_012, SCREEN_164: cmd.exe con hash modificato = rootkit). Può essere introdotto tramite side loading (SCREEN_690).
EXAM TIP — Rootkit: il file integrity monitoring è il rilevatore
Scenario: "FIM alert: hash di cmd.exe cambiato. Nessuna patch applicata negli ultimi 2 mesi" → Rootkit (SCREEN_012, SCREEN_164). Il rootkit modifica file di sistema critici. Senza patch che giustifichino la modifica, la causa è malware. Non è una collision crittografica (estremamente improbabile su file reali), non è una snapshot (non modifica il file).

Logic bomb (Bomba logica)

Una logic bomb (bomba logica) è codice malevolo dormiente che si attiva solo quando si verificano condizioni specifiche: una data, un'ora, l'assenza dell'utente dal sistema, il licenziamento di un account. Non è rilevabile finché le condizioni non si verificano perché sembra codice legittimo. È tipicamente piantata da un insider malevolo — un dipendente scontento o in procinto di essere licenziato — che vuole colpire l'organizzazione dopo la sua partenza.

Esempio pratico — Logic bomb da insider (OBJ 2.3 · SCREEN_440 + OBJ 2.4 · SCREEN_109)
Il codice: IF DATE() = "01/30/2023" THEN BEGIN DROP DATABASE WebShopOnline; END inserito nello script di backup da un DBA in uscita è una logic bomb classica: condizione = data specifica; payload = eliminazione del database. Appariva come codice legittimo per mesi, si attiva silenziosamente alla data programmata. Il primo passo investigativo: verificare i DBA recentemente licenziati.

Attacchi fisici (Physical Attacks)

Brute force fisico

Il brute force fisico nel contesto degli attacchi fisici si riferisce al tentativo di accedere fisicamente a un'area o a un dispositivo attraverso la forza: forzare serrature, rompere vetri, tagliare recinzioni. Da non confondere con il brute force informatico (tentativi di password): il contesto è l'accesso fisico non autorizzato tramite forza diretta.

RFID cloning (Clonazione RFID)

La clonazione RFID (RFID cloning) è la copia non autorizzata dei dati presenti in un badge o in una scheda di accesso RFID/NFC. Un lettore RFID portatile nascosto vicino al badge legittimo della vittima può copiare i dati del badge in pochi secondi, senza che la vittima se ne accorga. Il clone viene poi usato per accedere fisicamente alle aree protette come se fosse il badge originale.

L'indicatore caratteristico è il log di accesso fisico che mostra lo stesso badge usato in più luoghi in un intervallo di tempo impossibile (impossible travel) o in rapida sequenza sullo stesso varco (SCREEN_662: multiple entries from the same employee ID badge within a two-minute period).

EXAM TIP — RFID cloning: l'indicatore è la duplicazione temporale
"Multiple entries from an employee's ID badge within a two-minute period" → RFID cloning (SCREEN_662). Non è tailgating (che è una persona che segue fisicamente un'altra senza usare il badge). Non è shoulder surfing (che riguarda l'osservazione di informazioni visualizzate). La rapidità degli accessi sullo stesso badge indica che è stato clonato e usato da un'altra persona.

Environmental attacks (Attacchi ambientali)

Gli attacchi ambientali (environmental attacks) sfruttano o provocano condizioni ambientali avverse per causare danni ai sistemi IT: sbalzi di tensione, variazioni di temperatura, umidità eccessiva, interferenze elettromagnetiche, o danni fisici deliberati (allagamento, incendio, sabotaggio degli impianti di climatizzazione). Un data center senza sistemi di protezione ambientali adeguati (UPS, HVAC, sistema antincendio) è vulnerabile anche ad attacchi non informatici.

Attacchi di rete (Network Attacks)

DDoS — Amplified e Reflected

Un DDoS (Distributed Denial of Service) è un attacco che travolge un sistema bersaglio con traffico proveniente da molteplici sorgenti distribuite (botnet), rendendo il servizio inaccessibile agli utenti legittimi. L'indicatore primario è un aumento improvviso e massiccio del traffico in ingresso, spesso con caratteristiche anomale (stesse flag TCP, stesso payload, stesse porte sorgente).

Esempio — Lettura di log DDoS (SCREEN_304)
Log firewall: molti IP sorgente diversi, tutti inviando pacchetti SYN alla stessa destinazione (100.50.20.7:443) nello stesso secondo. Questo pattern — molti SYN senza ACK di risposta — è un SYN flood DDoS. Il server esaurisce le risorse mantenendo aperte connessioni TCP half-open in attesa degli ACK che non arriveranno mai.
Amplified DDoS (Amplificato)
L'attaccante invia piccole richieste a server intermedi (DNS, NTP, SSDP) con IP sorgente falsificato (quello della vittima). I server rispondono con risposte molto più grandi alla vittima. L'amplificazione può essere 50x–100x: 1 GB di traffico inviato dall'attaccante diventa 50–100 GB ricevuti dalla vittima.
Reflected DDoS (Riflesso)
L'attaccante usa server legittimi come "riflettori" inviando richieste con IP sorgente della vittima. I server rispondono alla vittima (non all'attaccante). La tecnica usa la spoofing dell'IP sorgente. Il DNS reflected DoS (SCREEN_264) usa il DNS come riflettore: piccola query DNS → grande risposta DNS verso la vittima.
EXAM TIP — DDoS reflected: l'indicatore DNS
"DNS server: CPU/memoria minimi, ma interfaccia di rete inondata di traffico in ingresso. Solo poche query DNS inviate da questo server" → Reflected DoS (SCREEN_264). Il server DNS non sta generando il traffico: lo sta ricevendo come riflettore. L'attaccante ha spoofato l'IP del DNS server e inviato richieste a molti server che ora rispondono tutti al DNS server (vittima).

DNS attacks (Attacchi DNS)

Il DNS poisoning (avvelenamento DNS) — anche detto DNS spoofing — inserisce record DNS falsi nella cache di un resolver DNS, reindirizzando gli utenti verso indirizzi IP controllati dall'attaccante invece di quelli legittimi. È particolarmente insidioso perché opera in modo trasparente per l'utente: l'URL digitato è corretto, ma viene risolto con un IP sbagliato.

L'indicatore caratteristico è la modifica non autorizzata dei record DNS (SCREEN_565: i record A e CNAME che puntano da 192.168.1.1 a 233.123.123.23 — un IP non appartenente all'azienda). La difesa è DNSSEC (DNS Security Extensions).

Il DNS tunneling (SCREEN_218) usa le query DNS come canale nascosto per esfiltrare dati: l'attaccante codifica i dati rubati nei sottodomain delle query DNS (es. dati.base64encoded.attacker.com). Il traffico DNS è spesso permesso anche dai firewall più restrittivi, rendendolo un canale di esfiltrazione efficace. Indicatore: volume elevato di query DNS verso domini inusuali durante orari non lavorativi.

Wireless attacks (Attacchi wireless)

Evil twin (Access point gemello malevolo)
Un attaccante crea un access point Wi-Fi con lo stesso SSID e caratteristiche di un AP legittimo. Gli utenti si connettono al falso AP credendolo legittimo. L'attaccante vede tutto il traffico non cifrato e può iniettare contenuti malevoli.
Deauthentication attack
L'attaccante invia frame di deauthentication falsificati per disconnettere i client dall'AP legittimo, forzandoli a riconnettersi (possibilmente all'evil twin) o per catturare l'handshake WPA2 durante la riconnessione (per attacchi offline alla password).
Jamming
Interferenza radio che rende inutilizzabile il canale wireless. Indicatore: degradazione o interruzione del segnale Wi-Fi in un'area specifica senza causa tecnica apparente.

On-path attack (Attacco sul percorso — MitM)

Un on-path attack (storicamente noto come Man-in-the-Middle, MitM) è un attacco in cui l'attaccante si posiziona segretamente tra due parti comunicanti, intercettando e potenzialmente modificando il traffico. La vittima crede di comunicare direttamente con la controparte legittima, ma tutto il traffico transita attraverso l'attaccante.

Lo scenario SCREEN_584 (attaccante che intercetta e riusa le credenziali di login per accedere a informazioni finanziarie) descrive un on-path attack con credential replay. Il pass-the-hash (SCREEN_418) è una variante: invece delle credenziali in chiaro, l'attaccante ruba e riusa l'hash NTLM dell'autenticazione.

Credential replay (Replay delle credenziali)

Un attacco di credential replay riusa credenziali di autenticazione catturate in precedenza per autenticarsi come la vittima senza conoscere la password originale. Include il pass-the-hash (uso dell'hash NTLM), il pass-the-ticket (uso di ticket Kerberos rubati — golden ticket, silver ticket), e il replay di token di sessione o cookie.

Il comando mimikatz sekurlsa::pth /ntlm:B4B9B02E... (SCREEN_418) è il tool più famoso per il pass-the-hash: estrae gli hash NTLM dalla memoria LSASS di Windows e li usa per aprire nuove sessioni autenticate senza conoscere la password.

EXAM TIP — Pass-the-hash: riconosci mimikatz
Se vedi mimikatz o sekurlsa::pth o /ntlm: in un comando → pass-the-hash attack (SCREEN_418). Mimikatz è lo strumento standard per il dump degli hash NTLM e il loro riuso. Non è social engineering, non è fileless malware.

Attacchi applicativi (Application Attacks)

Injection (Iniezione)

Gli attacchi di injection inseriscono dati malevoli in input che vengono poi interpretati dal sistema come comandi. La famiglia include: SQL injection (dati SQL → database), command injection (comandi OS → shell), LDAP injection (query LDAP → directory service), XML/XPath injection. Il denominatore comune è la mancanza di input validation. La mitigazione universale è sempre l'input validation (SCREEN_727).

Esempio — Data exfiltration via DNS tunneling (SCREEN_400)
Il PCAP mostra traffico da 10.5.1.1 verso 52.165.16.154 usando encapsulation IPv6 in IPv4 (proto 41) su una porta non standard (13788). Dati sensibili vengono esfiltrati incapsulando IPv6 dentro IPv4 per nascondere il traffico malevolo in quello apparentemente legittimo. L'indicatore: protocollo di encapsulation usato su porte non standard verso IP esterni sospetti.

Privilege escalation (Escalation dei privilegi)

La privilege escalation è l'atto di ottenere permessi superiori a quelli originariamente concessi. Esistono due varianti: verticale (da utente normale a amministratore/root) e orizzontale (da utente A ad utente B con lo stesso livello di privilegi ma accesso a risorse diverse). È quasi sempre un passo intermedio — raramente un fine in sé — nel percorso dell'attaccante verso il suo obiettivo finale.

Tecniche comuni
Sfruttamento di SUID/SGID bit su Unix, token impersonation su Windows, Kerberoasting (richiesta di ticket TGS per account con SPN e offline cracking dell'hash), DLL hijacking, unquoted service paths.
Valid accounts (SCREEN_516)
L'attaccante usa credenziali rubate o create appositamente per accedere a sistemi con privilegi elevati. È una tecnica MITRE ATT&CK: sfrutta account legittime invece di vulnerabilità tecniche, rendendo l'attività difficile da distinguere da quella normale.

Forgery (Falsificazione)

CSRF — Cross-Site Request Forgery (Falsificazione delle richieste tra siti)
Forza un browser autenticato a inviare richieste non volute a un sito web per cui l'utente è già autenticato. L'attaccante inserisce una richiesta malevola in una pagina che l'utente visita: il browser la invia automaticamente con i cookie di sessione dell'utente. Mitigazione: token CSRF nelle form, SameSite cookie attribute.
SSRF — Server-Side Request Forgery (Falsificazione lato server)
L'attaccante forza il server web a fare richieste verso risorse interne non accessibili dall'esterno (es. metadata endpoint del cloud, servizi interni). Il server diventa un proxy non intenzionale.

Directory traversal

Il directory traversal (path traversal) usa sequenze ../ per navigare fuori dalla directory consentita e accedere a file di sistema sensibili. Pattern canonico: GET ../../../../etc/passwd. Già trattato in OBJ 2.3 come tipo di vulnerabilità; qui è il suo indicator come attacco in uno scenario.

Attacchi crittografici (Cryptographic Attacks)

Downgrade · Collision · Birthday

Downgrade attack (Attacco di degradazione)
Forza due sistemi a usare una versione più vecchia e vulnerabile di un protocollo crittografico, invece di quella sicura negoziata originariamente. Esempio classico: POODLE (forzava da TLS a SSLv3), BEAST (forzava TLS 1.0). Indicatore: negoziazione di protocolli obsoleti (SSL, TLS 1.0) nonostante la disponibilità di versioni moderne.
Collision attack (Attacco di collisione)
Trova due input diversi che producono lo stesso hash (collisione). Se l'algoritmo di hashing è debole (MD5, SHA-1), l'attaccante può creare un file malevolo con lo stesso hash di un file legittimo: il sistema di verifica dell'integrità non rileva la sostituzione. Indicatore: due file con hash identico ma contenuto diverso.
Birthday attack (Attacco del compleanno)
Sfrutta il paradosso del compleanno della teoria della probabilità: con N possibili valori di hash, bastano √N input casuali per avere probabilità del 50% di trovare una collisione. È il fondamento matematico degli attacchi di collisione. Indica la necessità di hash più lunghi per resistere a questi attacchi.

Attacchi alle password (Password Attacks)

Password spraying

Il password spraying è un attacco che testa una singola password comune (o poche password) su molti account diversi. È l'opposto del brute-force classico (molte password su un account). L'obiettivo è evitare i lockout degli account: la maggior parte dei sistemi blocca un account dopo N tentativi falliti consecutivi sullo stesso account. Con lo spraying, ogni account riceve un solo tentativo — mai abbastanza per scatenare il lockout.

Esempio — Lettura dei log di password spraying (SCREEN_010, SCREEN_143)
Log: login fallito per administrator, jsmith, guest, cpolk, fmartin — tutti con la stessa password "Spring2023", negli stessi secondi. Pattern diagnostico del password spraying: password uguale, username diversi, timestamp ravvicinati. Non è brute-force (che userebbe password diverse sullo stesso account). Non è pass-the-hash (che userebbe hash, non password in chiaro).
Brute-force (informatico)
Testa tutte le combinazioni possibili di password su un singolo account finché non trova quella corretta. Molto lento, scatena facilmente i lockout. Indicatore: molti tentativi falliti consecutivi sullo stesso account (SCREEN_198: password authentication succeeded ma MFA failed ripetutamente = tentativo di brute-force sul MFA).
Differenza operativa chiave
Spraying: 1 password × N account → evita lockout. Brute-force: N password × 1 account → causa lockout. Il pattern nei log è diagnostico e immediato.

Pass-the-hash

Il pass-the-hash è un attacco di credential replay che usa l'hash NTLM di una password di Windows per autenticarsi, senza conoscere la password in chiaro. NTLM autentica usando un challenge-response basato sull'hash, non sulla password: se l'attaccante ha l'hash, può autenticarsi esattamente come se avesse la password. Lo strumento principale è mimikatz con il modulo sekurlsa::pth.

Indicatori di compromissione (Indicators of Compromise — IoC)

Gli Indicators of Compromise (IoC) sono segnali osservabili che suggeriscono che un sistema è stato compromesso o è sotto attacco. Non sono prove definitive, ma trigger per l'investigazione. Riconoscerli rapidamente riduce il dwell time (tempo che l'attaccante trascorre inosservato nel sistema).

Account lockout (Blocco account)
Un account viene bloccato dopo troppi tentativi di autenticazione falliti. Può indicare brute-force o password spraying. Anomalo se avviene per molti account contemporaneamente o per account normalmente non usati.
Concurrent session usage (Sessioni concorrenti)
Lo stesso account risulta autenticato da più posizioni geografiche o da più dispositivi contemporaneamente. Indica credenziali rubate usate da un attaccante mentre l'utente legittimo è ancora connesso.
Impossible travel (Viaggio impossibile)
Un account si autentica da Milano alle 10:00 e da Tokyo alle 10:05 — fisicamente impossibile. Indica credenziali rubate usate da una posizione geografica diversa da quella dell'utente legittimo. Molti SIEM moderni rilevano questo pattern automaticamente.
Resource consumption (Consumo di risorse anomalo)
CPU, memoria, I/O disco o larghezza di banda di rete significativamente superiori al normale senza causa giustificata. Indicatori tipici di: cryptojacking (CPU elevata), ransomware in fase di cifratura (I/O disco elevato), worm (traffico di rete elevato), DDoS in ingresso.
Resource inaccessibility (Inaccessibilità delle risorse)
Sistemi, servizi o file che normalmente sono disponibili diventano irraggiungibili. Indica potenziale DDoS, ransomware che ha cifrato i file, o sabotaggio (cancellazione, corruzione dei dati).
Out-of-cycle logging (Logging fuori ciclo)
Log generati in orari inusuali (es. attività amministrativa alle 3:00 di notte) o picchi di logging improvvisi durante orari di basso utilizzo. Indicatore di attività non autorizzata quando gli amministratori legittimi non sono presenti.
Missing logs (Log mancanti)
Un attaccante sofisticato cerca di coprire le proprie tracce cancellando o modificando i log. L'assenza di log in un periodo in cui dovrebbero esistere è un indicatore di compromissione (e spesso peggiore dei log anomali, perché significa che l'attaccante aveva privilegi sufficienti per cancellarli).
Blocked content (Contenuti bloccati)
Un sistema tenta di accedere a contenuti che vengono bloccati da filtri di sicurezza: URL categorizzate come malware, IP in blacklist, download di file eseguibili non autorizzati. Indica malware che tenta di scaricare payload aggiuntivi o di comunicare con C2.
Published/documented (Pubblicato/documentato)
Una vulnerabilità o un indicatore di compromissione che è stato pubblicamente documentato da vendor, CISA, o threat intelligence. Se un IoC è già nella threat intelligence, il sistema dovrebbe rilevarlo automaticamente.
Esempio pratico — Data exfiltration via VPN (SCREEN_586)
I log VPN mostrano che un utente si connette durante la notte (orari non lavorativi = out-of-cycle logging) e trasferisce dati verso un IP sospetto (resource consumption anomala + blocked content potenziale). La combinazione di orario anomalo + destinazione sospetta + volume di traffico = indicatori di data exfiltration. Il fatto che l'attività avvenga via VPN legittima suggerisce credenziali rubate.

Ransomware Indicators Clicca per girare

Presenza di file con estensioni anomale (.ryk, .locked, .crypto) + dicitura di riscatto (.txt/.html) + picchi elevatissimi di I/O disco causati dal processo di cifratura massivo dei file.

Trojan vs Worm propagation Clicca per girare

Trojan: richiede interazione o installazione cosciente (nascosto dentro software lecito).
Worm: si diffonde in modo 100% automatico e autonomo tra sistemi in rete senza interazione.

Rootkit vs Logic Bomb indicatori Clicca per girare

Rootkit: alterazione hash di sistema (alert FIM) + mascheramento processi avanzati.
Logic Bomb: codice letale silente attivato da date/orari o licenziamenti dipendenti.

DNS Poisoning vs DNS Tunneling Clicca per girare

DNS Poisoning: manipolazione dei record DNS per fare hijack traffico.
DNS Tunneling: esfiltrazione fraudolenta incapsulando dati crittografati in query/subdomain DNS.

Password Spraying vs Brute Force Clicca per girare

Spraying: prova la stessa password comune su migliaia di username (evita lockout).
Brute Force: prova migliaia di password sullo stesso singolo account utente.

RFID Cloning indicatori fisici Clicca per girare

Il medesimo badge ID registrato in varchi distanti nello stesso istante, oppure in varchi d'accesso consecutivi in un lasso di tempo fisicamente impossibile (impossibility travel locale).
